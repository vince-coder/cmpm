# CMPM 홈페이지 제작 계획 (plan.md)

> 토목 **전문**기업(중소기업) CMPM의 회사소개 홈페이지
> 도메인: `www.cmpm-group.com` (Vercel 연결 완료)
> 벤치마크: 삼성물산 건설부문 (https://www.samsungcnt.com/index.do)
> 컨셉: **매우 심플하지만 세련된** — 대기업 프레임을 중소기업 규모로 압축

---

## 1. 목표와 원칙

| 항목 | 방향 |
|------|------|
| 정체성 | **그라우팅 특화 · 특허전문 토목기업**. "기술력(특허 20+) · 신뢰 · 실적"을 핵심 메시지로 |
| 핵심 무기 | **차수 그라우팅 · 강관다단 그라우팅** 공법 + **보유 특허 20개 이상** → 사이트 전체가 "기술·특허"를 1순위로 강조 |
| 사업 | ① 그라우팅 시공(저수지·터널/지하철·철도·도로·교량·상하수도 적용) ② 교량 유지보수 ③ 상하수도 유지보수 |
| 톤앤매너 | 절제된 여백, 큰 타이포, 풀블리드 이미지 위 오버레이 텍스트 (삼성물산 차용) |
| 규모 | 페이지 5~6개 수준. 무거운 백엔드 없음(회사소개 사이트) |
| 반응형 | **모바일 우선** 설계 → 데스크톱 확장. 둘 다 1급 지원 |
| 운영 | 비개발자도 실적/공지를 큰 부담 없이 갱신 가능하도록 |

### 설계 철학 (삼성물산에서 가져올 "감각")
- **절제된 뉴트럴 팔레트 + 단일 액센트 컬러** (화려함 배제)
- **충분한 여백(whitespace)** 과 명확한 시각적 위계
- **고품질 대형 이미지 + 오버레이 타이포그래피**
- **숫자(실적 지표) 대시보드**로 신뢰 즉시 전달 ("AT A GLANCE")
- **대표 프로젝트 쇼케이스** ("LANDMARK")

## 2. 벤치마크 분석 — 무엇을 가져오고 무엇을 버릴까

삼성물산 건설부문 사이트는 대기업 규모라 그대로 따라가면 빈 껍데기가 됩니다.
**프레임(구조·디자인 언어)은 가져오되, 대기업 전용 기능은 과감히 버립니다.**

### ✅ 가져올 것 (Frame & Design)

| 삼성물산 요소 | CMPM 적용 |
|---------------|-----------|
| 회전 Hero 비주얼 + 슬로건 | 비주얼 1~3장 + "그라우팅 특화 · 특허 20+" 핵심 메시지 |
| **AT A GLANCE** (직원수·매출·국가수 지표) | **보유 특허 20+건** · 설립연도 · 누적 시공실적 건수 · 보유 면허/등급 |
| **LANDMARK** (대표 4대 프로젝트) | 대표 그라우팅 실적 3~6건 쇼케이스 (저수지·터널·교량 등) |
| 사업분야 탭(건축/토목/플랜트/주택) | **기술·특허**(차수/강관다단 그라우팅) + **사업분야**(그라우팅 적용·유지보수) 로 재구성 |
| 프로젝트 카드 캐러셀 | 실적 카드 그리드 + 상세 페이지 |
| 절제된 색·여백·타이포 | 동일한 디자인 언어 차용 |
| 명확한 GNB + Footer 구조 | 축소된 GNB(5개) + 간결한 Footer |

### ❌ 버릴 것 (대기업 전용 — SME엔 과함)

- **IR / 투자정보** (상장사 전용) — 제거
- **ESG 전용 대형 섹션** — 제거하거나 "회사소개" 안 한 단락으로 축소
- **글로벌 네트워크 47개국 지도** — 제거 (또는 "오시는 길" 단일 지도로 대체)
- **다부문 포트폴리오**(상사/패션/리조트) — 해당 없음
- **뉴스룸 고도 필터링** — 단순 "공지/소식" 리스트로 축소 (페이지는 **포함**)
- 다국어(KR/EN) — **한국어만으로 확정**, 추후 영문 확장 여지만 남김

GNB(상단 메뉴): **회사소개 · 기술·특허 · 사업분야 · 시공실적 · 문의**

│   └─ 오시는 길 (지도 + 주소)
├─ 기술·특허 (/technology)        ★ CMPM의 핵심 차별점 — GNB 최상위 강조

├─ 시공실적 (/projects)

## 4. 기술 스택 (Vercel 최적화 기준)

### 프론트엔드
| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 15 (App Router)** | Vercel 1st-party, SSG/SSR, 이미지 최적화, SEO 최강 |
| 언어 | **TypeScript** | 유지보수성·안정성 |
| 스타일 | **Tailwind CSS v4** | 빠른 반응형, 일관된 디자인 토큰 |
| UI 컴포넌트 | **shadcn/ui** (선택) | 접근성 좋은 기본 컴포넌트, 디자인 커스터마이즈 자유 |
| 애니메이션 | **Framer Motion** | 스크롤 페이드/슬라이드, Hero 전환 — "세련됨"의 핵심 |
| 폰트 | **Pretendard (한글) + Inter (영문)** | 삼성물산 *SamsungOne*의 인상을 거의 동일하게 재현하면서 상업용 자유(OFL) — 라이선스 리스크 0. (※ SamsungOne은 삼성 전용 브랜드 폰트라 타사 사용 불가) |
| 이미지 | **next/image** | Vercel 자동 최적화(WebP/AVIF, lazy) |
| 지도 | **Kakao Map** embed | 국내 사용자 친화 (오시는 길/문의) |

### 백엔드 (최소한 — 회사소개 사이트)
| 기능 | 선택 | 이유 |
|------|------|------|
| 폼 처리 | **Next.js Server Actions / Route Handler** | 별도 서버 불필요, Vercel 서버리스 |
| 이메일 발송 | **Resend** | 문의 폼 → 회사 이메일. 설정 간단, Vercel 친화 |
| 스팸 방지 | Honeypot + (선택)**Cloudflare Turnstile** | 무료, 비침습적 |
| 콘텐츠 관리 | **로컬 데이터(MDX/JSON) + GitHub 관리** (확정) | 실적/특허/공지를 코드로 관리, Git 푸시 → Vercel 자동 배포 |
| 분석 | **Vercel Analytics** + GA4 | 방문 통계 |

> **결론:** "회사소개 사이트"이므로 DB/로그인/결제 같은 무거운 백엔드는 **불필요**.
> 동적 요소는 ① 문의 폼 이메일 발송, ② 실적/공지 콘텐츠 갱신 — 이 둘뿐.

## 5. 콘텐츠 관리 방식 — **A. 로컬 데이터 + GitHub (확정)**

실적·특허·공지를 **코드 저장소(MDX/JSON)** 로 관리하고, **GitHub 푸시 → Vercel 자동 배포** 흐름으로 갱신합니다.

- **장점:** 무료, 빠름, 단순, Git 버전관리, Vercel과 완벽 연동(푸시 즉시 배포)
- **갱신 방법:** `/content` 폴더의 데이터 파일(JSON/MDX) 수정 후 GitHub에 커밋·푸시
- **데이터 구조 예시**
  - `content/patents.json` — 특허 목록(번호·명칭·등록일·이미지)
  - `content/projects/*.mdx` — 시공실적(공종·발주처·공기·사진)
  - `content/news/*.mdx` — 공지/소식
- **추후 확장:** 회사 직원이 웹에서 직접 편집할 필요가 커지면 Sanity 등 Headless CMS로 마이그레이션 (Next.js 구조상 용이)

## 6. 디자인 시스템

### 컬러 (절제된 뉴트럴 + 단일 액센트)
```
배경      : White (#FFFFFF) / Light Gray (#F5F6F7)
텍스트    : Charcoal (#1A1A1A) / Gray (#6B7280)
주조색    : Deep Navy / Charcoal (헤더·푸터·큰 면)
액센트    : Deep Blue (확정) — 신뢰·기술·안정감. 예) #1B3A6B ~ #1E4F91 계열
```
> 액센트 **딥 블루 1개만** 사용 → 삼성물산식 절제미. 정확한 HEX는 로고(CI) 확인 후 미세조정.

### 타이포그래피 (삼성물산 SamsungOne 인상 재현)
- 한글: **Pretendard** — 헤드라인 SemiBold/Bold, 본문 Regular/Medium
- 영문·숫자: **Inter** — 지표(AT A GLANCE)·영문 슬로건 강조 디스플레이
- 헤드라인: 크고 굵게, 넉넉한 줄간격 / 본문: 16px+ 가독성 우선
- CSS는 `next/font/local`(Pretendard) + `next/font/google`(Inter)로 셀프호스팅 → 빠른 로딩·CLS 방지

### 레이아웃 패턴
- 풀블리드 이미지 + 오버레이 텍스트 (Hero, 사업분야)
- 카드형 그리드 (실적)
- 좌우 교차 배치 (이미지/텍스트 alternating)
- 12-column 그리드, 넉넉한 여백

### 반응형 (모바일 ↔ 데스크톱)
- **모바일**: 햄버거 메뉴, 1-column 스택, 터치 타깃 최적화, 지표 2×2
- **태블릿**: 2-column
- **데스크톱**: 풀 GNB(상단 고정), 멀티 컬럼, 호버 인터랙션
- Breakpoints: `sm 640 / md 768 / lg 1024 / xl 1280`

### 마이크로 인터랙션
- 스크롤 진입 시 페이드/슬라이드업 (Framer Motion)
- Hero 비주얼 부드러운 크로스페이드
- 카드 호버 시 미세 확대·그림자

## 7. 페이지별 구성 상세

### 홈 (/)
1. **Hero** — 회전 비주얼 + 슬로건("그라우팅 특화 · 특허 20+") + CTA(기술보기/문의)
2. **AT A GLANCE** — **보유 특허 20+** · 설립연도 · 누적 실적 · 보유면허 (아이콘+숫자)
3. **핵심 기술·특허** — 차수/강관다단 그라우팅 + 특허 보유 강조 (→ /technology) ★
4. **사업분야 요약** — 그라우팅 시공 / 교량·상하수도 유지보수 카드 (→ /business)
5. **대표 실적(LANDMARK)** — 그라우팅 실적 3~6건 쇼케이스 (→ /projects)
6. **회사 소식** — 최근 공지 2~3건 (→ /news)
7. **CTA 밴드** — "프로젝트 문의하기" / 오시는 길

### 회사소개 (/about)
Hero 배너 → 인사말 → 경영이념/비전 → 연혁(타임라인) → **인증·면허** → 오시는 길(지도)

### 기술·특허 (/technology) ★ 핵심 페이지
Hero → **차수 그라우팅**(공법 원리·강점·적용 다이어그램) → **강관다단 그라우팅**(동일 구성) → **보유 특허 쇼케이스**(20+건 리스트·특허증 이미지·등록일) → 관련 실적 링크

### 사업분야 (/business)
Hero → 섹션: **그라우팅 시공**(적용분야: 저수지/터널·지하철/철도/도로/교량/상하수도) / **교량 유지보수** / **상하수도 유지보수** → 각 분야 설명+대표 이미지+관련 실적 링크

### 시공실적 (/projects, /projects/[slug])
- 목록: 필터(공종·연도·발주처) + 카드 그리드
- 상세: 프로젝트명·개요·위치·발주처·공기·규모·갤러리

### 문의 (/contact)
- 문의 폼(이름/연락처/이메일/제목/내용) → Resend 이메일 발송
- 회사 연락처·주소·이메일 · Kakao Map

## 8. 비기능 요구사항

- **SEO**: `next/metadata`, sitemap.xml, robots.txt, Open Graph, 한글 메타, 구조화 데이터(Organization)

- **보안**: 폼 스팸 방지, 환경변수로 API 키 관리(Vercel Env)
- **분석**: Vercel Analytics + GA4
